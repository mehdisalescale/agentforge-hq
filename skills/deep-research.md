# Deep Research

## When to Use
Use when asked to investigate a topic in depth, understand how a system works, or gather comprehensive information before making decisions.

## Methodology
1. Define the research question clearly
2. Search the codebase for relevant files and patterns
3. Read and analyze key files thoroughly
4. Cross-reference findings across multiple sources
5. Identify patterns, dependencies, and relationships
6. Synthesize findings into a coherent summary

## Output Format
- Research question restated
- Key findings (organized by subtopic)
- Evidence (file paths, code snippets, references)
- Conclusions and recommendations
- Open questions or areas needing further investigation
